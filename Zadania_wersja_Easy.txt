ZADANIE 2 (na 3.0) – “2FA light” (kod SMS/Email na sztywno)
Szkielet (o co chodzi)

Masz dodać drugą warstwę logowania, ale w najprostszej możliwej wersji:

po zalogowaniu hasłem użytkownik ma wpisać kod 123456

jeśli wpisze zły kod → nie wpuszczamy

To nie jest “prawdziwe 2FA”, ale pokazuje ideę i przepływ.

Wykonanie (krok po kroku)
Część A: Backend – dodaj prostą weryfikację kodu

Otwórz plik logowania w backendzie (zwykle):

backend/src/routes/auth.js (albo podobny)

Znajdź endpoint logowania, np.:

POST /login

Zrób tak, aby backend wymagał pola twoFactorCode:

jeżeli twoFactorCode !== "123456" → zwróć błąd

Przykład logiki (do wklejenia w endpoint logowania):

po sprawdzeniu hasła dopisz:

const { twoFactorCode } = req.body;
if (twoFactorCode !== "123456") {
  return res.status(401).json({ message: "Nieprawidłowy kod 2FA" });
}

Część B: Frontend – dodaj pole na kod

Otwórz ekran logowania:

frontend/src/.../Login... (np. Login.jsx / Login.vue)

Dodaj input:

“Kod 2FA (wpisz 123456)”

Upewnij się, że w request do backendu leci:

twoFactorCode

✅ Oddanie: screen z logowania (pole kodu) + screen, że z błędnym kodem nie wpuszcza, a z 123456 wpuszcza.

ZADANIE 3 (na 4.0) – “Lepsze szyfrowanie” w wersji EASY = szyfrowanie tylko jednej rzeczy
Szkielet (o co chodzi)

Wpisy vault mają pola (np. password, note).
Twoim zadaniem jest zrobić minimalne “lepsze bezpieczeństwo”:

zaszyfruj pole password przed zapisaniem do bazy

odszyfruj je przy odczycie

Wersja EASY: szyfrujesz tylko jedno pole (np. password) i wystarczy, że działa.

Wykonanie (krok po kroku)

Otwórz backend route dla wpisów, np.:

backend/src/routes/vault.js

Znajdź endpoint dodawania wpisu:

POST /entries lub podobny

Tam, gdzie zapisujesz wpis, zrób:

passwordEncrypted = encryptText(password)

do bazy zapisuj password: passwordEncrypted

Znajdź endpoint pobierania wpisów:

GET /entries

Przy zwracaniu danych zrób:

passwordDecrypted = decryptText(entry.password)

zwróć odszyfrowane hasło do frontu

Jeśli projekt już ma utils/crypto.js, to korzystasz z istniejących funkcji encryptText() i decryptText().

✅ Oddanie: screen wpisu przed/po + screen w MongoDB (że w bazie nie jest czysty tekst).

ZADANIE 4 (na 5.0) – Anti-bruteforce EASY = blokada po 5 błędnych logowaniach (na 1 minutę)
Szkielet (o co chodzi)

Specjalista IT zabezpiecza logowanie przed zgadywaniem haseł:

jak ktoś 5 razy poda złe hasło → blokada na minutę

Wersja EASY: licznik trzymasz w pamięci serwera (prosty obiekt JS).

Wykonanie (krok po kroku)

W backendzie (np. auth.js) dodaj na górze pliku:

const attempts = {}; 
// attempts[email] = { count: 0, blockedUntil: 0 }


W endpoint logowania:

weź email

sprawdź, czy jest blockedUntil > Date.now() → zwróć błąd “Spróbuj później”.

Jeśli hasło błędne:

zwiększ count

jeśli count >= 5 ustaw blockedUntil = Date.now() + 60*1000 (1 minuta)

Jeśli logowanie poprawne:

wyczyść licznik (delete attempts[email])

✅ Oddanie: screen: 5 błędnych prób → blokada → po minucie znowu można.

"scripts": {

npm test
